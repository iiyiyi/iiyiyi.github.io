<Dijkstra> POJ 1062 Expensive Betrothal

2020-12-30 • Category: Shortest Paths • Tag: Dijkstra

Link:poj.org/problem?id=1062

Brief Description

You would like to trade an item $I_i$ with a person. You can either pay $P_i$ for this item, or you can give this person an item $I_j$ and pay $S_{ji}$ for this item.
Every person you trade with have a rank $R_i$. If you trade with some one with rank $n$, then you cannot trade with any person whoe rank is grater than $n + m$.
How much money at least you have pay to buy item $I_1$?

Algorithm Analysis

First, we construct a node $v_0$. Consider each item as a node $v_i$. Link an edge from $v_0$ to each $v_i$. The length of the edge between $v_0$ and $v_i$ is $P_i$.
Then link an edge from $v_i$ to $v_j$ if we can give a person item $I_j$ and pay $S_{ji}$ to buy $I_i$. The length of the edge between such two nodes is $S_{ji}$.
Since we have constraints on the ranks, we can enumerate each node $v_i$ and assume that the highest rank in this trade is $R_i$. Hence, we cannot pass any node with rank outside the range
$[R_i - m, R_i]$. Then run a dijkstra to find the shortest path from $v_0$ to $v_1$.
