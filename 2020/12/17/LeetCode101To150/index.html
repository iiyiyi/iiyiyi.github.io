$\star$ Problem 127: Word Ladder

https://leetcode.com/problems/word-ladder/
Related Topics: BFS, Bidirectional BFS
Algorithm Analyais: the idea is that for each node representing a word abcdef..., we construct nodes representing
*bcdef..., a*cdef..., ab*def..., and so on. Then we add an edge between the original node and each of the generated nodes. Then implement dfs to find the answer.

Problem 146: LRU Cache

https://leetcode.com/problems/lru-cache/
Methond 1: In my solution, a hash table is used to store the key and the value. The only problem is how to dea with the LRU key when the number of keys exceeds the capacity.
My idea is to push the key $k$ into a queue when it is used and keep recording how many such $k$s currently exist in the queue. When deleting the LRU key, delete all keys that appear later in the queue. The remaining front must be
the LRU key.

Method 2: The official solution is to write a hash table and a doubly-linked list by ourself.

$\star\star$Problem 159: Max Points on a Line

https://leetcode.com/problems/max-points-on-a-line/
Related Topics: Map, gcd
Algorithm Analysis: The main idea is that for each point $i$, we would like
to find the maximum number of points on a line containing $i$. To do so, we calculate the slope
between $i$ and any other point $j$ and put the slope as a key into the map. To ensure the accuracy of the slope,
we calculate $dx$ and $dy$ and divide both of them by $gcd(dx, dy)$.
Notes: Note that their is no hash function for pair<int, int>, so we use
unordered_map<string, int> instead of unordered_map<pair<int, int>, int>. Also note
that an easy way to update the map is map[key]=value.
