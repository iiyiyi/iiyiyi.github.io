<Binary Search>Nearest Cities

2020-01-09 • Category: Binary Search • Tag: Binary Search

Link:https://algo.monster/problems/nearest_cities

Brief Description

There are multiple cities in an ordinary Cartesian plane. Each city has a name and is located at an integral (x, y) coordinate intersection. Write an algorithm to determine the name of the nearest city that shares either an x or a y coordinate with the queried city. If no other cities share an x or y coordinate, return NONE. If two cities have the same distance to the queried city, q[i], consider the one with an alphabetically smaller name (i.e. ‘ab’ < ‘aba’ < ‘abb’) as the closest choice.

Algorithm Analysis

Use two unordered_map<int, vector<int>>s to store all the cities that shares the same x or y coordinate. Each time when we quried the nearest city, we can binary search on each unordered map.
