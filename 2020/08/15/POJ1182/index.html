<Weighted Union-find Set> POJ1182 Food chain

2020-08-15 • Category: Data Structure • Tag: Union-find Set

Link: http://poj.org/problem?id=1182

Brief Description

There are three kinds of animals: $A$ eats $B$, $B$ eats $C$ and $C$ eats $A$. There are $1\le N\le 50000$ animals and $0\le K\le 100000$ messages stating either ‘1 X Y’(X and Y are the same animal), or ‘2 X Y’(X eats Y). A message is false if and only if:
(1) It contradicts with previous true messages;
(2) Either $X>N$ or $Y>N$;
(3) It states that $X$ eats $X$.

Algorithm Analysis

There are two approaches to this problem.
The first approach is similar to POJ1703. For an animal $i$, $i$ reperesents the case that $i$ is $A$; $i+N$ represents the case that $i$ is $B$; $i+2N$ represents the case that $i$ is C. Two elements belonging to the same union-find set means that they can be true at the same time.

In the approach above, we can observe that some cases are symmetric: $(i=A, j=B)$, $(i=B, j=C)$ and $(i=C, j=A)$ are actually the same. Therefore, we don’t actually need $3*N$ space.

The second approach is weighted union-find set. For $i$, weight[i] reperesents the relation between $i$ and the root of the union-set it belongs to: weight[i] == 0 means that $i$ and its root are the same animal; weight[i] == 1 means that $i$ eats its root; weight[i] == 2 means that $i$ is eaten by its root.
Obviously, for $i$ and $j$ belonging to the same union-find set, the relation between $i$ and $j$ is $(i-j+3)%3$. If the relation between the relaion between $i$ and $j$ is $r$, $weight[j] = (weight[i] + r) % 3$.
